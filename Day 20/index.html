Day 20: HTML Tables

Why Use Tables?

HTML tables should be used for tabular data only - not for page layout! Use them for:

  • Financial data
  • Comparison charts
  • Timetables
  • Statistics
  • Product specifications

üß© Table Elements

Core Elements

  • <table> - Table container
  • <caption> - Table title/description
  • <thead> - Header content
  • <tbody> - Body content
  • <tfoot> - Footer content
  • <tr> - Table row
  • <th> - Table header cell
  • <td> - Table data cell

Advanced Elements

  • colspan - Merge columns
  • rowspan - Merge rows
  • <colgroup> - Column grouping
  • <col> - Column properties
  • scope - Accessibility attribute

Accessibility Tips

  • Always use <th> for headers
  • Add scope attribute (col, row, colgroup, rowgroup)
  • Include <caption> for table description
  • Use aria-describedby for complex tables
  • Provide headers attribute for complex headers
  • Ensure proper color contrast

üí™ Practice Exercise

Create a Product Comparison Table

Requirements:

  1. Compare 3 products (Smartphone X, Tablet Y, Laptop Z)
  2. Include these columns: Feature, Product 1, Product 2, Product 3
  3. Include these rows: Price, Screen Size, RAM, Storage, Battery Life, Camera
  4. Use colspan for a "Best Value" row
  5. Use rowspan if any feature applies to multiple products
  6. Add proper accessibility features:
    • <caption> with description
    • scope attributes
    • aria-describedby if needed
  7. Style the table with:
    • Striped rows
    • Hover effects
    • Different header background
    • Responsive design

Starter Code Structure:

<div class="responsive-table">
  <table>
    <caption>...</caption>
    <thead>
      <tr>
        <th scope="col">Feature</th>
        <!-- Add product headers -->
      </tr>
    </thead>
    <tbody>
      <tr>
        <th scope="row">Price</th>
        <!-- Add price data -->
      </tr>
      <!-- Add more rows -->
      <tr>
        <td colspan="4"><strong>Best Value: Tablet Y</strong></td>
      </tr>
    </tbody>
  </table>
</div>
