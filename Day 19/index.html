What are Semantic Elements?

Semantic HTML elements clearly describe their meaning to both browser and developer.

<header>

Introductory content or navigation aids. Typically contains headings, logos, search forms. 

<header>
  <h1>Website Title</h1>
  <p>Tagline</p>
</header>

<nav>

Section with navigation links. Use for major navigation blocks only. 

<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/about">About</a></li>
  </ul>
</nav>

<main>

Primary content of the document. Only one per page.

<main>
  <!-- Primary content here -->
</main>

<section>

Thematic grouping of content, typically with a heading.

<section>
  <h2>Chapter 1</h2>
  <p>Content...</p>
</section>

<article>

Self-contained composition that could be distributed independently. 

<article>
  <h2>Blog Post Title</h2>
  <p>Blog content...</p>
</article>

<aside>

Content indirectly related to main content (sidebars, pull-quotes). 

<aside>
  <h3>Related Links</h3>
  <ul><li>...</li></ul>
</aside>

<footer>

Footer for its nearest sectioning content or root element.

<footer>
  <p>¬© 2024 Company Name</p>
</footer>

üåü Benefits of Semantic HTML

Before (Div Soup)

<div id="header">
  <div class="logo">Logo</div>
  <div class="nav">...</div>
</div>

<div id="main-content">
  <div class="post">...</div>
</div>

<div id="bottom">
  <div class="copyright">...</div>
</div>

Problems:

  • Search engines can't understand structure
  • Screen readers lack context
  • Hard to maintain
  • No built-in meaning

After (Semantic HTML)

<header>
  <img src="logo.png" alt="Logo">
  <nav>...</nav>
</header>

<main>
  <article>...</article>
</main>

<footer>
  <p>Copyright...</p>
</footer>

Benefits:

  • Better SEO (search engines understand content)
  • Improved accessibility (screen readers)
  • Easier to style and maintain
  • Future-proof and standard compliant

    </section>
    
    <section aria-labelledby="specs">

    </section>

      <!-- Related products list -->
    </aside>
  </article>
</main>

‚ôø Accessibility Benefits

Semantic HTML provides built-in accessibility features:

  • Screen readers can navigate by landmarks (header, nav, main, footer)
  • Keyboard navigation works better with semantic structure
  • ARIA roles are often unnecessary when using proper semantics
  • Browser defaults provide better styling and behavior
  • Search engines understand content hierarchy for better ranking

Landmark Roles (Automatic with Semantic Elements):

  • <header> ‚Üí role="banner"
  • <nav> ‚Üí role="navigation"
  • <main> ‚Üí role="main"
  • <footer> ‚Üí role="contentinfo"
  • <aside> ‚Üí role="complementary"
