XHTML 1.0 Strict

XHTML (Extensible HyperText Markup Language) is HTML reformulated as XML. It enforces strict syntax rules for well-formed documents.

Strict Rules:

  • All elements must be closed
  • All tags must be lowercase
  • Attributes must be quoted
  • Proper nesting required
  • No attribute minimization
XHTML Document Example Strict Syntax 
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN"
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html xmlns="http://www.w3.org/1999/xhtml" lang="en">
<head>
    <title>XHTML Example</title>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
</head>
<body>
    <h1>Welcome</h1>
    <p>This is <strong>important</strong> text.</p>
    <br />  <!-- Self-closing tag -->
    <img src="image.jpg" alt="Description" />
</body>
</html>

XHTML Advantages:

  • Strict validation catches errors early
  • Compatible with XML tools and parsers
  • Clean, predictable structure
  • Better for automated processing

HTML5 Modern Standard

HTML5 introduces semantic elements, multimedia support, and flexible syntax while maintaining backward compatibility.

Flexible Rules:

  • Optional closing tags for many elements
  • Case-insensitive (but lowercase recommended)
  • Unquoted attributes allowed (but quoted recommended)
  • Forgiving parser handles errors gracefully
  • New semantic elements (header, nav, section, etc.)
HTML5 Document Example Modern & Flexible 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>HTML5 Example</title>
</head>
<body>
    <header>
        <h1>Welcome</h1>
    </header>
    
    <nav>
        <a href="#">Home</a>
    </nav>
    
    <main>
        <article>
            <p>This is <strong>important</strong> text.</p>
            <br>  <!-- No slash needed -->
            <img src=image.jpg alt=Description>
        </article>
    </main>
</body>
</html>

HTML5 Advantages:

  • Backward compatible
  • New semantic elements
  • Native multimedia support
  • Built-in form validation
  • Better accessibility features

üé® XML in HTML: SVG (Scalable Vector Graphics)

SVG is an XML-based vector image format that can be embedded directly in HTML5. It's resolution-independent and can be styled with CSS.

Basic Shapes

Interactive SVG

SVG Embedded in HTML XML-based Vector Graphics

üí™ Practice Exercise

Create a Modern Scientific Web Page

Requirements:

  1. XHTML-Strict Document:
    • Create a valid XHTML 1.0 Strict document
    • Include proper XML declaration
    • Use strict syntax rules (all tags closed, lowercase, quoted attributes)
    • Add mathematical content using presentational markup
  2. HTML5 Modern Document:
    • Convert the same content to HTML5
    • Use semantic elements (header, nav, main, article, footer)
    • Embed SVG graphics for scientific diagrams
    • Use MathML for equations (or CSS-styled alternatives)
    • Add multimedia elements (video explaining concepts)
  3. Content to Include:
    • Scientific article about a topic (physics, math, biology)
    • At least 3 mathematical equations
    • 2 SVG diagrams (graphs, molecular structures, etc.)
    • Interactive elements (buttons to show/hide derivations)
    • Responsive design that works on mobile
  4. Technical Requirements:
    • Proper DOCTYPE for both versions
    • Character encoding set to UTF-8
    • Viewport meta tag for responsive design
    • Accessibility features (alt text, ARIA labels)
    • Clean, well-commented code
