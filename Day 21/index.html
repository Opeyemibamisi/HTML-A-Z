Day 21: Unordered Lists & Attributes Mastery

Why Lists Matter

Unordered lists (<ul>) are essential for structuring content. When combined with id and class attributes, they become powerful tools for both styling and JavaScript manipulation.

Key Concepts:

  • id - Unique identifier (one per page)
  • class - Reusable identifier (multiple elements)
  • data-* - Custom data attributes
  • CSS can target elements by ID, class, or attributes

The id attribute provides a unique identifier for an element. Use it for:

  • Anchor links (href="#section1")
  • JavaScript DOM manipulation
  • CSS styling (with #id-name selector)
  • Form label association (for="input-id")

This element has id="unique-element"

Notice the yellow background? That's CSS targeting this specific ID:

#unique-element {
    background: #fff3cd;
    border: 2px dashed #ffc107;
    padding: 20px;
}

ID Best Practices:

  • IDs must be unique on the page
  • Start with a letter (not a number)
  • Use kebab-case: main-header, not mainHeader
  • Avoid special characters except hyphens and underscores

ðŸŽ¨ The class Attribute

Classes are for reusable styling and grouping. Multiple elements can share the same class.

Multiple Classes

Elements can have multiple classes separated by spaces:

This paragraph has class="highlight success"

This paragraph has class="highlight warning"

<p class="highlight success">...</p>
<p class="highlight warning">...</p>

.highlight { background: #ffeaa7; }
.success { color: #2ecc71; }
.warning { color: #e74c71; }

Class Selectors in CSS

  • .class-name - targets elements with that class
  • element.class - targets specific elements with class
  • .class1.class2 - targets elements with BOTH classes

Common Class Naming Conventions:

  • BEM: .block__element--modifier
  • kebab-case: .main-navigation
  • Descriptive: .text-center, .bg-primary
